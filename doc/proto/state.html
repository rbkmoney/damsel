Thrift module: state

Определения структур и сервисов для поддержания взаимодействия со state processor – абстракции, реализующей шаг обработки (другими словами, один переход состояния) ограниченного конечного автомата со сложным состоянием, которое выражается при помощи истории как набора событий, порождённых процессором.

Struct: ComplexAction

KeyFieldTypeDescriptionRequirednessDefault value
1set_timerSetTimerActionoptional
2tagTagActionoptional

Желаемое действие, продукт перехода в новое состояние. Возможные действия представляют собой ограниченный язык для управления прогрессом автомата, основанием для прихода сигналов, которые приводят к дальнейшим переходам состояния. Отсутствие заполненных полей будет интерпретировано буквально, то есть автомат перестанет прогрессировать в отсутствии внешних вызовов.

Struct: SetTimerAction

KeyFieldTypeDescriptionRequirednessDefault value
1timerbase.TimerКритерий остановки таймера ожидания required

Действие установки таймера ожидания на определённый отрезок времени. По истечению заданного отрезка времени в процессор поступит сигнал `TimeoutSignal`.

Struct: TagAction

KeyFieldTypeDescriptionRequirednessDefault value
1tagbase.TagЗначение для ассоциации required

Действие ассоциации с процессом автомата произвольного значения После факта успешной ассоциации к автомату можно обратиться с внешним вызовом, используя указанное значение, то есть в процессор может поступить вызов `processCall` с указанным `tag` в качестве `Reference`.

Services

Service: Processor

Процессор переходов состояния ограниченного конечного автомата. В результате вызова каждого из методов сервиса должны появиться новое состояние и новые действия, приводящие к дальнейшему прогрессу автомата.

Function: Processor.processSignal

SignalResult processSignal(SignalArgs a)
Обработать поступивший сигнал.

Function: Processor.processCall

CallResult processCall(CallArgs a)
Обработать внешний вызов и сформировать ответ на него.
